1. 在source中找到对应的js, 并保存到本地, 然后对其修改
2. 本地创建一个空文件夹, 名字随意
3. 在空文件夹中创建和js目录结构一模一样的文件夹结构, 例如 `g.alicdn.com/assets-group/pc-sem/1.0.156/pages/home/index.js`, 把修改的js放到该目录下
4. source左侧打开Overrides, 选中第二部的空文件夹, 需要同意一下, 然后可以看到激活状态的图表
5. 然后刷新页面即可成功执行被修改后的js

## 存储

爬到的数据,  经常会有新增需求的情况, 过几天说还要页面上的其他字段, 为了不重新爬, 只能将页面保存下来, 但是如果是html, 数据库存储空间根本不够, 只能存到对象存储中. 为了节约对象存储的空间和带宽, 可以压缩后存储, 压缩方式最好是gzip这种浏览器支持的, 那么可以直接在浏览器上查看, 在存储对象存储时添加Content-Encoding=gzip header即可.

public List<String> decode(String filePath) {
  try {
      QRCodeMultiReader formatReader = new QRCodeMultiReader();
      BufferedImage bufferedImage = ImageIO.read(new File(filePath));
      BinaryBitmap binaryBitmap =
              new BinaryBitmap(new HybridBinarizer(new BufferedImageLuminanceSource(bufferedImage)));
      // 定义二维码参数
      Map hints = new HashMap<>(1);
      hints.put(EncodeHintType.CHARACTER_SET, "utf-8");
      com.google.zxing.Result[] result = formatReader.decodeMultiple(binaryBitmap, hints);
      return Arrays.stream(result).map(Result::getText).toList();
  } catch (NotFoundException e) {
      return Collections.emptyList();
  } catch (Exception e) {
      log.error("Unknown exception in zxing recognition of QR code:{}", filePath, e);
      return Collections.emptyList();
  }
}
