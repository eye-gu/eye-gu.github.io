服务提供方重启的时候, 消费者几乎每次都会报这个错.

## dubbo 3.2.10

查看dubbo3.2.10代码`org.apache.dubbo.remoting.transport.netty4.NettyConnectionClient#doClose`

```java
@Override
protected void doClose() {
    // AbstractPeer close can set closed true.
    if (isClosed()) {
        if (LOGGER.isDebugEnabled()) {
            LOGGER.debug(String.format("Connection:%s freed ", this));
        }
        final io.netty.channel.Channel current = getNettyChannel();
        if (current != null) {
            current.close();
        }
        this.channel.set(null);
        closePromise.setSuccess(null);
    }
}
```

原因很清晰, closePromise的setSuccess被调用了两次, 那么就是doClose被调用了两次



一次是`org.apache.dubbo.rpc.protocol.tri.TripleProtocol#destroy`这里, invoker调用destroy然后调用协议的destroy.

还有一次是`org.apache.dubbo.remoting.transport.netty4.NettyConnectionClient#initBootstrap`, 利用的是netty的closeFuture机制, 在初始化Channel的时候配置的. 两者并发调用的时候就会出现该异常
