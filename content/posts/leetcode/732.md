### 差分数组

从小到大遍历, 遇上正数, 说明后续的时间点都被预订了一次, 遇上负数, 说明预订结束了, 后面的时间点没有被预订.

很容易理解, 但是需要从头遍历计算最大的预订.

```rust
use std::collections::BTreeMap;

struct MyCalendarThree {
    cnt: BTreeMap<i32, i32>
}

impl MyCalendarThree {

    fn new() -> Self {
        MyCalendarThree { cnt: BTreeMap::new() }
    }

    fn book(&mut self, start_time: i32, end_time: i32) -> i32 {
        *self.cnt.entry(start_time).or_insert(0) += 1;
        *self.cnt.entry(end_time).or_insert(0) -= 1;
        let mut max = 0;
        let mut current = 0;
        for (&k,&v) in &self.cnt {
            current += v;
            max = max.max(current);
        }
        max
    }
}
```



### 线段树

首先用idx来构建树, id为idx的树其左子树为`2*idx`, 右子树为`2*idx+1`

id为1的树, 其范围为0-1000000000, 左子树id为2, 范围为0-500000000, 右子树id为3, 范围为500000001-1000000000. 以此类推.

tree的key为树的id, value为树范围内最大的重复预订数量, 那么显然等于其左右子树的最大预订数量.

lazy不好理解, 举个例子就是有个范围为100-120的树, 预订范围也是100-120, 如果没有lazy, 那么还需要继续递归, 把左子树100-110, 右子树111-120的最大预订数量也加上1. 有了lazy后, 就直接在100-120树上标记这整个区间都被预订了, 不需要继续递归子树, 所以子树的最大预订需要加上lazy的数量, 才是这个区间的最大预订数量.

```rust
use std::collections::HashMap;

struct MyCalendarThree {
    tree: HashMap<i32, i32>,
    lazy: HashMap<i32, i32>
}

impl MyCalendarThree {

    fn new() -> Self {
        MyCalendarThree {
            tree: HashMap::new(),
            lazy: HashMap::new()
        }
    }

    fn book(&mut self, start_time: i32, end_time: i32) -> i32 {
        self.update(start_time, end_time-1, 0, 1000000000, 1);
        *self.tree.get(&1).unwrap_or(&0)
    }

    fn update(&mut self, start:i32, end:i32, l:i32, r:i32, idx:i32) {
        if r < start || end < l {
            return;
        }
        if start <= l && r <= end {
            self.tree.insert(idx, self.tree.get(&idx).unwrap_or(&0) + 1);
            self.lazy.insert(idx, self.lazy.get(&idx).unwrap_or(&0) + 1);
        } else {
            let mid = (r + l) / 2;
            self.update(start, end, l, mid, 2 * idx);
            self.update(start, end, mid + 1, r, 2 * idx + 1);
            self.tree.insert(idx, self.lazy.get(&idx).unwrap_or(&0) + self.tree.get(&(2 * idx)).unwrap_or(&0).max(self.tree.get(&(2*idx+1)).unwrap_or(&0)));
        }
    }
}
```
