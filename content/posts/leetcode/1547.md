https://leetcode.cn/problems/minimum-cost-to-cut-a-stick/description/



这动态规划的状态定义和转移方程也太难想到了.



```java
class Solution {
    public int minCost(int n, int[] cuts) {
        int m = cuts.length;
        Arrays.sort(cuts);
        int[] newCuts = new int[m + 2];
        newCuts[0] = 0;
        for (int i = 1; i <= m; ++i) {
            newCuts[i] = cuts[i - 1];
        }
        newCuts[m + 1] = n;
        m += 2;
        // f[i][j]表示newCuts[i]到newCuts[j]木棍完全切好的最小成本
        // f[i][i+1]不需要切
        // f[i][i+2]只能在newCuts[i+1]处切
        int[][] f = new int[m][m];
      	// 遍历方式确保了从小往大
      	// 第一次是m-3,m-5, 就是在m-4处切一刀
        // 第二次是m-2,m-4, 在m-3处切一刀
        // 第三次是m-2,m-5, 要不在m-3处切一刀, 此时m-3和m-5已经在第一次计算得到, m-2和m-3不需要切, 默认0
      	//                 要不在m-4处切一刀, 此时m-2和m-4已经在第二次计算得到, m-4和m-5不需要切, 默认0
        for (int i = m-3; i >= 0; --i) {
            for (int j = i+2; j < m; ++j) {
                f[i][j] = Integer.MAX_VALUE;
                for (int k = i+1; k < j; ++k) {
                    f[i][j] = Math.min(f[i][j], f[i][k] + f[k][j]);
                }
                f[i][j] += newCuts[j] - newCuts[i];
            }
        }
        return f[0][m-1];
    }
}
```
