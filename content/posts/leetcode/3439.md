https://leetcode.cn/problems/reschedule-meetings-for-maximum-free-time-i/description/


```java
class Solution {
        public int maxFreeTime(int eventTime, int k, int[] startTime, int[] endTime) {
            int n = startTime.length;
            // 所有会议的间隙
            int[] gaps = new int[n + 1];
            // 第一个间隙就是0到第一个会议的开始时间
            gaps[0] = startTime[0];
            for (int i = 0; i < n - 1; i++) {
                gaps[i + 1] = startTime[i + 1] - endTime[i];
            }
            // 最后一个间隙就是最后一个会议的结束时间到eventTime
            gaps[n] = eventTime - endTime[n - 1];
            int ans = 0;
            for (int i = 0; i < n + 1 - k; i++) {
                // 计算所有相邻的k+1个空隙的和
                int sum = 0;
                for (int j = 0; j<=k; j++) {
                    sum += gaps[i + j];
                }
                ans = Math.max(ans, sum);
            }
            return ans;
        }
    }
```

超时了, 因为时间复杂度是O(n*k), 很容易想到使用前缀和来优化内部的循环, 不过循环的边界好难想, 测试了半天.

```java
class Solution {
        public int maxFreeTime(int eventTime, int k, int[] startTime, int[] endTime) {
            int n = startTime.length;
            // preSumGaps[i] 代表从0到i的空隙的和
            int[] preSumGaps = new int[n + 1];
            preSumGaps[0] = startTime[0];
            for (int i = 0; i < n - 1; i++) {
                preSumGaps[i + 1] = preSumGaps[i] + startTime[i + 1] - endTime[i];
            }
            preSumGaps[n] = preSumGaps[n - 1] + eventTime - endTime[n - 1];
            // preSumGaps[k]即[0,k]的空隙之和
            int ans = preSumGaps[k];
            for (int i = 0; i < n - k; i++) {
                // preSumGaps[i + k + 1] - preSumGaps[i]即[i+1,i+1+k]的空隙和
                ans = Math.max(ans, preSumGaps[i + k + 1] - preSumGaps[i]);
            }
            return ans;
        }
    }
```
